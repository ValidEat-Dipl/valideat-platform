## HR-/Admin-Plattform

Der Desktop-Prototyp umfasst zwölf Screens:

1. Login
2. Register
3. Übersicht
4. Zuletzt erstellte Markerl
5. Markerl hinzufügen
6. Markerl-Detail
7. Markerl korrigieren
8. Markerl-Korrektur prüfen
9. Clearing
10. Clearing-Konflikt
11. Konflikte
12. Export

## Mobile Webanwendung für Mitarbeitende

Der mobile Prototyp umfasst neun Screens:

1. Welcome
2. Login
3. Register
4. Start
5. Verwendung erstellen
6. Angabe prüfen
7. Daten erfasst
8. Letzte Erfassungen
9. Erfassungsdetail

Beide Bereiche werden über dieselbe Figma-Datei dokumentiert. Getrennte Repository-Dateien sind deshalb aktuell nicht notwendig.
